# FRDM-MCXN947 Zephyr RTOS Windows 安装教程

本文用于在 Windows 中安装 Zephyr RTOS 开发环境，并验证 FRDM-MCXN947 能够成功编译 Zephyr 示例。

Zephyr 官方文档：

```text
https://docs.zephyrproject.org/latest/develop/getting_started/index.html
```

FRDM-MCXN947 官方 Zephyr 板卡文档：

```text
https://docs.zephyrproject.org/latest/boards/nxp/frdm_mcxn947/doc/index.html
```

Zephyr GitHub 仓库：

```text
https://github.com/zephyrproject-rtos/zephyr
```

## 1. 打开 PowerShell

建议使用 Windows PowerShell 或 Windows Terminal。

检查当前 PowerShell 是否为管理员：

```powershell
if (([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) { "管理员" } else { "非管理员" }
```

普通用户一般也可以安装 Zephyr。若安装工具时权限不足，再以管理员身份打开 PowerShell。

## 2. 安装基础工具

先检查 `winget`：

```powershell
winget --version
```

安装 Zephyr 需要的基础工具：

```powershell
winget install Kitware.CMake Ninja-build.Ninja oss-winget.gperf Python.Python.3.12 Git.Git oss-winget.dtc wget 7zip.7zip
```

安装完成后，关闭 PowerShell，重新打开，再执行：

```powershell
cmake --version
ninja --version
python --version
git --version
dtc --version
```

如果 `dtc --version` 无法识别，需要重新检查 Devicetree Compiler 是否安装成功，或者把 `dtc.exe` 所在目录加入系统 `PATH`。

## 3. 创建 Zephyr 目录和 Python 虚拟环境

本文把 Zephyr 安装到 `D:\Zephyr`：

```powershell
mkdir D:\Zephyr
cd D:\Zephyr
python -m venv zephyrproject\.venv
```

激活虚拟环境：

```powershell
zephyrproject\.venv\Scripts\Activate.ps1
```

如果命令行前面出现 `(.venv)`，说明虚拟环境已经启用。

如果 PowerShell 提示脚本执行被禁止，可以执行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后重新激活：

```powershell
zephyrproject\.venv\Scripts\Activate.ps1
```

## 4. 安装 west

`west` 是 Zephyr 的工程管理和构建工具。

```powershell
pip install west
west --version
```

## 5. 下载 Zephyr 源码和模块

进入工作目录：

```powershell
cd D:\Zephyr\zephyrproject
```

初始化 Zephyr：

```powershell
west init -m https://github.com/zephyrproject-rtos/zephyr --mr main
```

下载 Zephyr 依赖模块：

```powershell
west update
```

这一步会比较久，也比较依赖网络。成功后会出现：

```text
D:\Zephyr\zephyrproject\zephyr
```

## 6. 导出 Zephyr CMake 包

```powershell
west zephyr-export
```

这个命令会把 Zephyr 注册到当前用户的 CMake 包注册表中，方便后续工程自动找到 Zephyr。

## 7. 安装 Zephyr Python 依赖

```powershell
python -m pip install @((west packages pip) -split ' ')
```

## 8. 安装 Zephyr SDK

FRDM-MCXN947 使用 ARM Cortex-M 内核，因此只需要安装 ARM 工具链：

```powershell
cd D:\Zephyr\zephyrproject\zephyr
west sdk install --toolchain arm-zephyr-eabi
```

如果下载 SDK 时出现 GitHub rate limit，可以使用自己的 GitHub token：

```powershell
west sdk install --toolchain arm-zephyr-eabi --personal-access-token YOUR_TOKEN_HERE
```

注意：不要使用教程、视频或他人文档里的 token，也不要把自己的 token 发给别人。

## 9. 检查 FRDM-MCXN947 板卡支持

搜索 MCXN947：

```powershell
west boards | Select-String -Pattern "mcxn947" -CaseSensitive:$false
```

应能看到类似板卡名：

```text
frdm_mcxn947/mcxn947/cpu0
```

后续构建 FRDM-MCXN947 工程时，使用这个板卡名。

## 10. 构建 Hello World 示例

先用 Zephyr 自带示例验证环境：

```powershell
cd D:\Zephyr\zephyrproject\zephyr
west build -p always -b frdm_mcxn947/mcxn947/cpu0 samples\hello_world
```

如果编译成功，说明 Zephyr 基础环境已经可用。

也可以构建 LED 闪烁示例：

```powershell
west build -p always -b frdm_mcxn947/mcxn947/cpu0 samples\basic\blinky
```

## 11. 烧录程序

使用 USB 线连接 FRDM-MCXN947，然后执行：

```powershell
west flash
```

FRDM-MCXN947 通常使用板载 MCU-Link CMSIS-DAP，Zephyr 官方文档中的默认 runner 是 `linkserver`。

如果 `west flash` 提示找不到 LinkServer，需要安装 NXP 的 LinkServer 或 MCUXpresso IDE 相关工具。

## 12. 查看串口输出

烧录 `hello_world` 后，可以打开串口工具查看输出。

常用串口参数：

```text
115200 baud
8 data bits
No parity
1 stop bit
```

输出类似：

```text
Hello World! frdm_mcxn947/mcxn947/cpu0
```

## 13. 查看版本信息

查看 Zephyr 版本：

```powershell
cd D:\Zephyr\zephyrproject
git -C zephyr describe --tags
git -C zephyr describe --tags --abbrev=0
```

查看 west 版本：

```powershell
west --version
```

查看 Zephyr SDK 版本：

```powershell
type D:\Zephyr\zephyrproject\zephyr\SDK_VERSION
```

## 14. 后续用于温湿度传感器工程

W71 教程中的 DHT11 温湿度工程属于 Zephyr sensor 示例。迁移到 FRDM-MCXN947 时，应用层 `main.c` 通常不需要大改，重点是新增 FRDM-MCXN947 的设备树 overlay。

构建温湿度工程时，板卡名仍然使用：

```powershell
west build -p always -b frdm_mcxn947/mcxn947/cpu0 <你的温湿度工程目录>
```

DHT11 的 DATA 引脚接到 FRDM-MCXN947 的哪个 GPIO，后续就需要在 overlay 中写对应的 `dio-gpios`。
